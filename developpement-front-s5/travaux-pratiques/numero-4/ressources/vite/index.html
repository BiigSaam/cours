Consignes de la pratique - TP4 - Vitejs

Partie 1 (cliquer pour afficher / cacher)
  1. Créez votre premier projet vitejs grâce à la commande npm create vite@latest _nom_de_dossier_

    Durant la mise en place du projet vite, pensez à sélectionner un projet "Vanilla" et utiliser le langage "Javascript".

  2. Installez les dépendances avec npm install (dans le dossier crée par vite)
  3. Lancez le projet via la commande npm run dev
  4. Observez un peu le code
    • Mettez à jour le code HTML
    • Ajoutez un nouveau fichier CSS (et du contenu dans le fichier) puis importez-le dans le fichier main.js

Partie 2 (cliquer pour afficher / cacher)
  1. Installez sass dans le projet avec la commande npm install -D sass à la racine de votre projet

    Le paramètre "-D" (ou --save-dev) indique que votre dépendance node n'est utile que dans l'environnement de développement. Et donc n'a pas à être dans le build final de notre application.

  2. Changez l'extension d'un de vos fichiers .scss en .css
  3. Utilisez les fonctionnalités de SCSS comme l'imbrication de style (nesting)
    • Voir documentation de SCSS sur l'imbrication de style (ouverture dans un nouvel onglet)
  4. Exportez votre projet pour la production avec la commande npm run build

Partie 3 (cliquer pour afficher / cacher)

Par défaut, VSCode ne gère pas les fichiers .twig, il faudra installer un plugin gratuit twig via le système d'extensions. Et pour avoir l'auto-complétion HTML dans les fichiers .twig, il faudra modifier les réglages.

  1. Allez dans le menu File > Preferences > Settings
  2. Recherchez "emmet" dans la barre de recherche
  3. Dans la zone "Emmet: Includes languages", cliquez sur "Add Item" et mettez "twig" dans la zone "key" et "html" dans la zone "value"
  1. Ouvrez la console dans le dossier "vite-twig" et installez les dépendances avec la commande npm install

    Pour ouvrir directement votre dossier dans la console. Vous pouvez le glisser dans VSCode puis aller dans le menu Terminal > New Terminal

  2. Essayez une des fonctionnalités

L'API retourne une liste.

Pour aller plus loin (cliquer pour afficher / cacher)
    • Groupez les jours fériés par année
    • Affichez le contenu de chaque année avec les balises <details> et <summary>
    • Proposez à l'utilisateur de sélectionner les différentes régions et mettez à jour les informations en fonction

Pensez à mettre les données en cache pour éviter un appel d'api à chaque changement de région. La liste des régions est trouvable dans la documentation. Accéder à la documentation (ouverture dans un nouvel onglet)
