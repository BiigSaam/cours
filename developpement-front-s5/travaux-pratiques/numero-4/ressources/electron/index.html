Consignes de la pratique - TP4 - Electron

Partie 1 (cliquer pour afficher / cacher)
  1. Installez electron via nodejs
    • Accéder au démarrage rapide d'electron

    Il est préférable d'avoir la dernière version LTS de Node d'installée sur votre ordinateur. Si vous avez besoin de gérer plusieurs versions de Nodejs sur votre ordinateur, vous pouvez utiliser un outil comme nvm.

    Installer nvm

  2. Créez votre première application (voir exemple plus bas sur le lien précédent)

Partie 2 (cliquer pour afficher / cacher)
  1. Créez une communication (IPC) entre le main process et le renderer process.
    • Pour cette étape, ceci peut être un console.log() comme dans l'exemple. A noter que le console.log() dans le main process s'affiche dans la console de node et non celle de votre application
  2. Créez une autre communication (IPC) entre le main process et le renderer process, retournez le résultat et affichez-le dans votre application.
  3. Créez une troisième communication (IPC) entre le main process et le renderer process. Le main process doit récupérer la liste des fichiers dans le dossier de votre choix et être affichée dans le renderer process.
    • Il est préférable d'utiliser le package node globby pour récupérer la liste des fichiers
      • Accéder à la documentation de globby
  4. Pour aller plus loin : affichez la liste des fichiers grâce à la balise <template>
  • Accéder à la documentation de IPC (ouverture dans un nouvel onglet)

Astuce : Pour éviter d'ouvrir la console du navigateur via le menu View > Toggle Developper Tools ou Ctrl / Cmd + maj + i, vous pouvez rajouter la ligne suivante win.webContents.openDevTools() dans la fonction CreateWindow (celle qui contient la fonction "loadFile()")

Astuce 2 : Vous avez dû le remarquer, vous devez arrêter votre application puis la relancer à chaque mise à jour de code, c'est fastidieux. Pour vous éviter ceci, il existe le node module "electron-reload" qui recharge votre application à chaque modification. Télécharger electron-reload (ouverture dans un nouvel onglet)

Partie 3 (cliquer pour afficher / cacher)
  1. Créez un fichier texte (.txt) dans votre dossier de travail
  2. Lisez le contenu de ce fichier grâce à nodejs depuis votre main process et affichez-en le contenu dans votre renderer process
  3. Modifiez le contenu du fichier et renvoyez la réponse vers le main process puis sauvegardez le fichier texte sous un autre nom

Note : Si vous appelez votre évènement dès la création de votre application, il ne sera jamais intercepté par le renderer process. Deux solutions :

  • Mettre l'appel d'évènement dans un setTimeout(() => {})
  • Appeler votre évènement depuis une API native du système d'exploitation. Par exemple, ajouter une nouvelle entrée dans le menu principal de votre application avec le code suivant dans la promesse de "app.whenReady": 
      const mainMenu = [
      // Contient les données actuelle du menu principal
      ...Menu.getApplicationMenu().items,
      {
        label: "Main Process",
        submenu: [
          {
            // Nouvelle entrée de menu
            label: "Évènement",
            click: () => {
              win.webContents.send("my_event", "hello world");
            },
          },
        ],
      },
  ];
  
  const menu = Menu.buildFromTemplate(mainMenu);
  Menu.setApplicationMenu(menu);

Partie 4 (cliquer pour afficher / cacher)
  1. Etablissez une communication permettant d'effectuer un appel d'API
  2. Affichez le résultat dans le main renderer
  • Si vous n'avez pas d'API à exploiter sous la main, vous pouvez reprendre un de celles proposées dans le TP3 en S4. Télécharger le TP3 (ouverture dans un nouvel onglet)

N'oubliez pas que vous êtes dans un environnement Nodejs, vous pouvez tout à fait séparer vos fichiers puis les importer via des modules. Cependant pour des questions de sécurité, ceci ne fonctionenra pas dans votre preload process dans le renderer process, il faudra utiliser des outils comme Parcel, vite ou webpack pour séparer votre preload process.

Dans la ressource, il y a une config vite, pour l'utiliser, il faut installer vite via npm et la commande npm install -D vite et rajoutez la commande vite build dans l'objet "scripts" de votre fichier package.json. Et le fichier vite.config.js au même niveau que votre fichier package.json.

Dans les grandes lignes, la config vite surveille les modifications sur les fichers javascript et compile le tout dans un dossier dist/. Assurez-vous que les noms de vos scripts collent bien avec ceux de la configuration. Tout comme pensez bien à changer les chemins de fichiers pour qu'ils chargent les fichiers dans le dossier dist/.

  1. Vous aurez besoin d'installer les dépendances suivantes via npm :
    • npm install vite --save-dev
    • npm install @rollup/plugin-node-resolve --save-dev
  2. Ajouter une tâche npm executant la commande "vite". Ceci va lancer un serveur

Note : Il existe des configurations clé en main avec vite et electron. Comme electron-vite et plein d'autres. Rien ne vous empêche d'utiliser un de ses outils, toutefois, ils ont leur propre architecture de fichiers ce qui vous obligera à déplacer les vôtres pour la respecter.

Partie 5 (cliquer pour afficher / cacher)
  1. Créez un fichier CSS et ajoutez un peu de couleurs à votre application electron
  • L'utilisation de bibliothèques CSS externes comme tailwindcss est possible. Leur utilisation est identique à ce que vous avez appris jusqu'à présent.

Si vous souhaitez utiliser tailwindcss, l'outil vite s'y prête bien. Il vous suffit juste de suivre le didacticiel proposé par tailwindcss pour le configurer avec vite. Et ensuite de charger votre css dans le javascript. Ce fichier sera compilé à chaque sauvegarde.

  • Accéeder au didacticiel tailwindcss + vite (ouverture dans un nouvel onglet)

Partie 6 (cliquer pour afficher / cacher)
  1. Affichez une notification native lorsque votre chargement d'API est terminé avec le contenu que vous souhaitez.
    • Accéder à la documentation de Notification (ouverture dans un nouvel onglet)
    • Pour éviter que la notification s'affiche dès que vous faites l'appel d'API, vous pouvez ralentir l'exécution avec une promesse. Vous pouvez utiliser le code suivant :

      const delay = ms => new Promise(resolve => setTimeout(resolve, ms));

Note : Si vous êtes sous Windows, votre notification aura pour en-tête "electron.app.Electron", c'est le nom par défaut de l'application. Normalement, lorsque vous faites un build, c'est le nom de l'application qui apparaît. Vu que nous sommes toujours en dev, nous allons devoir forcer le nom via un peu de code.
  1. Dans le fichier package.json, rajoutez à la racine de l'objet, la clé "productName" avec la valeur souhaitée.
  2. Dans le fichier de départ, rajoutez le bloc suivant :
      if (process.platform === 'win32')
{
      app.setAppUserModelId(app.name);
}
