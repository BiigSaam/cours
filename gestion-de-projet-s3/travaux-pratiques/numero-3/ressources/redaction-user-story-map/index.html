Contexte de l'exercice (cliquer pour afficher / cacher)

On a vu précédemment que les Stories (et leurs tests d'acceptation) sont très utiles. Toutefois, elles se placent dans un ensemble qui peut être représenté par une User Story Map. Il permet d'organiser les Stories selon une chronologie qui permet d'avoir une vue d'ensemble du produit. De ce fait, selon le produit, cette Story Map peut être très dense aussi bien verticalement qu'horizontalement.

  • Guide - Comment découper une Story (ouverture dans un nouvel onglet)

Gardez votre progression, ça vous servira pour d'autres TP.

Consignes de la pratique

Partie 1 (cliquer pour afficher / cacher) - Liste des epics

Reprenez vos Stories du TP précédent et placez-les dans le cadre d'une Story Map. Il vous faudra certainement rajouter de nouvelles features (1 à 3) ainsi que les epics / stories. Pensez bien à ordonner vos features / epics dans l'ordre chronologique, et les stories par priorité. Rappel des epics proposés précédemment :

  • Gérer les cours à l'IUT (pour les étudiants)
  • Créer un compte sur un forum
  • Système d'achat de contenus additionnels pour un jeu vidéo (type de jeu au choix)
  • Rechercher une offre d'emploi
  • Gestion du panier (sur un site d'e-commerce)
  • Système de pagination
  • Gestion d'un agenda

Pour réaliser votre Story Map, vous pouvez utiliser un outil gratuit et collaboratif comme Trello (accéder au site de Trello)
