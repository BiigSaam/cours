Contexte de l'exercice (cliquer pour afficher / cacher)

Parfois, il arrivera qu'une (User) Story soit trop grosse pour être gérée durant un sprint, elle ne respecte donc pas le "S" d'INVEST (Independant, Negociable, Valuable, Estimable, Small et Testable pour rappel), dans ce cas, on parlera d'epic (ou épopée), et vous devrez découper cet epic en plusieurs (User) Stories. Ceci est le but de cet exercices. Parmi, les cinq epics proposées dans la partie 1, listez (seul ou à plusieurs), autant d'(User) Stories associées.

Pour rappel, une (User) Story, il est préférable qu'elle ait le format suivant :

« En tant que [persona], je veux [quoi] afin de [pourquoi] »

  • Guide - Comment découper une Story (ouverture dans un nouvel onglet)

Consignes de la pratique

Partie 1 (cliquer pour afficher / cacher) - Liste des epics (un au choix)

  • Gérer les cours à l'IUT (pour les étudiants)
  • Créer un compte sur un forum
  • Gestion du panier (sur un site d'e-commerce)
  • Rechercher une offre d'emploi

Partie 2 (cliquer pour afficher / cacher) - Position absolute

  1. Ajoutez une nouvelle balise (vous pouvez copier la balise section ayant la classe "positionnement")
    • Arrangez-vous pour que la balise ait une balise après pour bien voir les effets
  2. Retirez toute classe différente de "positionnement" (si vous avez copié la balise section du début)
  3. Mettez cette nouvelle balise en positionnement absolue (utilisez une classe)
  4. Au sein de la même classe jouez sur les propriétés "top", "bottom", "left" ou "right" pour déplacer la balise

    • Voir documentation de la propriété top (ouverture dans un nouvel onglet)
    • Voir documentation de la propriété bottom (ouverture dans un nouvel onglet)
