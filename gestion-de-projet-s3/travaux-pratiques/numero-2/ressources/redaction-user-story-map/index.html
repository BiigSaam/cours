Contexte de l'exercice (cliquer pour afficher / cacher)

On a vu précédemment que les Stories (et leurs tests d'acceptation) sont très utiles. Toutefois, elles se placent dans un ensemble qui peut être représenté par une User Story Map. Il permet d'organiser les Stories selon une chronologie qui permet d'avoir une vue d'ensemble du produit. De ce fait, selon le produit, cette Story Map peut être très dense aussi bien verticalement qu'horizontalement.

  • Guide - Comment découper une Story (ouverture dans un nouvel onglet)

Gardez votre progression, ça vous servira pour d'autres TP.

Consignes de la pratique

Partie 1 (cliquer pour afficher / cacher) - Liste des epics

En groupe, réalisez votre morning routine sous forme de User Story Map. Dans ce cas, nous pouvons déjà dégager au minimum trois gros thèmes :

  • Réveil
  • Hygiène
  • Départ

A vous rajouter ceux restants ainsi que les epics et user stories. Il n'est pas utile de respecter le gabarit "en tant que..." pour les user stories, un titre suffit, bien évidemment ça doit représenter une tâche atomique.

Pour réaliser votre Story Map, vous pouvez utiliser un outil gratuit et collaboratif comme Trello (accéder au site de Trello) ou les post-it fournis. Essayez d'avoir un code couleur entre vos thèmes/features, epic et story map

Partie 2 (cliquer pour afficher / cacher) - Liste des epics

A partir d'un des thèmes suivant, réalisez son user story map. Pensez bien à ordonner vos features / epics dans l'ordre chronologique, et les stories par priorité. Il n'est pas utile de respecter le gabarit "en tant que..." pour les user stories, un titre suffit, bien évidemment ça doit représenter une tâche atomique.

  • Répondre à un message sur un forum - L'utilisateur n'est pas connecté à la base
  • Réservation d'un voyage (moyen de transport et hébergement inclus)
  • Achat d'un produit sur un site d'e-commerce jusqu'au paiement
