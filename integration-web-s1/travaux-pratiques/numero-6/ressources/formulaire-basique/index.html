Le but de cette pratique est de construire notre premier formulaire en y ajoutant au fur et à mesure des éléments de formulaire tels que la balise <input /> ou encore <select>. Les formulaires sont des éléments clés du web 2.0 et 3.0, grâce à eux, et aux serveurs, l'utilisateur peut communiquer avec le site ou même d'autres internautes. Ainsi, il est promordial d'apporter beaucoup de soin à la construction des formulaires pour éviter de nuire à l'expérience de vos internautes. Sur un bon site, un internaute prend toujours du plaisir à y revenir.

Pré-consignes de la pratique - facultatives (cliquer pour afficher / cacher)

  1. Créez un nouveau fichier html (avec la structure de base d'un site web. Rappel, vous avez le raccourci "html:5")
  2. Copiez la balise <main> (et son contenu de ce fichier) dans le fichier que vous venez de créer (si elle existe)
  3. Importez le fichier css présent dans le dossier "css/" dans votre fichier de pratique
  4. Importez les fichiers js présents dans le dossier "js/" dans votre fichier de pratique. Voir plus bas dans le code de ce fichier pour un exemple ou même copier les lignes de code (les placer avant </body>
  5. Sauvegardez-le et commencez votre TP dedans

Consignes de la pratique

Partie 1 (cliquer pour afficher / cacher)
  1. Ajouter une balise <form> avec la valeur "" pour l'attribut "action"
  2. Ajouter une balise <input> avec son label en prenant soin de rendre le tout accessible, et y entrer une valeur. La balise <input> doit avoir la valeur "text" pour l'attribut "type"
  3. Ajouter dans le même formulaire un autre champ input avec un type "intelligent" (exemple <input type="email">) et mettre une valeur incorrecte. Ne pas oublier le label associé avec l'attribut "for"
    • Voir documentation de la balise <label> (ouverture dans un nouvel onglet)
    • Voir documentation de la balise <input type="text"> (ouverture dans un nouvel onglet)
    • Voir documentation de la balise <form> (ouverture dans un nouvel onglet)

Partie 2 (cliquer pour afficher / cacher)
  • Rajoutez un élément de formulaire permettant sa soumission
    • Note : pensez bien à remplir votre formulaire
  • Mettez une valeur ("get" ou "post") pour la propriété "method"
  • Soumettez le formulaire
    • Si target="GET" regardez l'url du navigateur, vous verrez les données dedans après soumission
    • Si target="POST" regardez l'onglet "console" de la console du navigateur ctrl + alt + i ou F12 ou "Clic droit" > "Inspecter / Examiner l'élément"

Partie 3 (cliquer pour afficher / cacher)
  1. Rajouter une balise <textarea> avec le label associé et les bons attributs
    • Voir documentation <textarea> (ouverture dans un nouvel onglet)
  2. Rajouter des inputs type radio (min. 2)
    • Voir documentation des boutons radio (ouverture dans un nouvel onglet)
  3. Rajouter des inputs type checkbox
    • Voir documentation des cases à cocher (ouverture dans un nouvel onglet)

Partie 4 (cliquer pour afficher / cacher)
  1. Rajouter une liste déroulante simple
    • Voir documentation des cases à cocher (ouverture dans un nouvel onglet)
    • Accéder à un site pour pour générer une liste rapidement (ouverture dans un nouvel onglet)

Partie 5 (cliquer pour afficher / cacher)
  1. Rajoutez l'attribut "required" sur l'un des champs ajouté précédemment (plusieurs si vous le souhaitez)
  2. Appliquez une pseudo-classe ":required" dans le CSS et appliquer des déclarations CSS pour identifier les champs requis
  3. Soumettez le formulaire sans remplir les champs requis

Partie 6 (cliquer pour afficher / cacher)
  1. Indiquez clairement les <label> dont les champs sont requis

    • Version simple :

      Rajoutez un pseudo-élément "::before" ou "::after" pour ajouter un "*" après tous les labels dont le champ est requis grâce à une classe associée à ces <label>
      • Voir documentation du pseudo élément ::before (ouverture dans un nouvel onglet)
      • Voir documentation du pseudo élément ::after (ouverture dans un nouvel onglet)

    • Version avancée :

      1. Jouez sur la propriété "order" des enfants flexbox ou la valeur "row-reverse" de la propriété "flex-direction"
      2. Jouez sur les combinateurs de voisin
        • Voir documentation du pseudo élément ::before (ouverture dans un nouvel onglet)
        • Voir documentation du pseudo élément ::after (ouverture dans un nouvel onglet)
        • Voir documentation du combinateur de voisin direct (ouverture dans un nouvel onglet)
